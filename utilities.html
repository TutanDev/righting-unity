Guides

MessageBus

Type-safe N:M pub/sub for decoupled architecture. Messages are unmanaged struct — no GC heap allocation, no delegate equality problems. Built for 72–120 Hz XR frame budgets.

Define a message

A message is any unmanaged struct implementing IMessage. The unmanaged constraint guarantees no managed references — no string, no arrays, no class fields.

Messages.cs
using Utilities.Messaging;

public struct PlayerDied : IMessage
{
    public int PlayerId;
    public float Timestamp;
}

// If you need string-like data, use FixedString:
public struct SceneLoaded : IMessage
{
    public int SceneIndex;
    public FixedString64Bytes SceneName;
}

Subscribe and unsubscribe

Subscribe returns a SubscriptionToken — an integer handle. Store it and pass it to Unsubscribe. No delegate equality comparison, no allocation on subscription changes.

Subscriber.cs
private SubscriptionToken _token;

void OnEnable()
{
    _token = MessageBus.Instance.Subscribe<PlayerDied>(OnPlayerDied);
}

void OnDisable()
{
    MessageBus.Instance.Unsubscribe(_token);
}

void OnPlayerDied(ref PlayerDied msg)
{
    Debug.Log($"Player {msg.PlayerId} died at {msg.Timestamp}");
}

Publish and enqueue

Publish dispatches immediately on the calling thread. Enqueue is thread-safe — use it from worker threads. Call DrainQueues() on the main thread to process queued messages.

Publisher.cs
// Immediate dispatch (main thread)
var msg = new PlayerDied { PlayerId = id, Timestamp = Time.time };
MessageBus.Instance.Publish(ref msg);

// Thread-safe enqueue (any thread)
MessageBus.Instance.Enqueue(in msg);

// Drain on main thread (e.g., in Update)
MessageBus.Instance.DrainQueues();

Timer

Handle-based, pool-backed deferred callback scheduler. No string-based Invoke, no coroutine allocations. Pre-allocated pool of 64 slots (configurable). Main-thread only.

Setup

Attach TimerHost to your persistent GameObject. Optionally create a TimerSettings ScriptableObject to configure pool size. Access via Timer.Instance.

Scheduling callbacks

TimerUsage.cs
using Utilities.Timing;

// One-shot after 2 seconds
TimerHandle handle = Timer.Instance.Schedule(2f, () => SpawnEnemy());

// Repeating every 5 seconds
TimerHandle loop = Timer.Instance.ScheduleRepeating(5f, () => CheckAI());

// Custom params: unscaled time, first fire after 1s, then every 0.5s
var p = TimerParams.Repeat(0.5f).WithDelay(1f).WithUnscaledTime(true);
TimerHandle precise = Timer.Instance.Schedule(p, () => Tick());

// Cancel a timer (safe — uses generation counter)
Timer.Instance.Cancel(handle);

TimerParams fluent API

Method / PropertyDescription
TimerParams.Default1.0s delay, one-shot, scaled time
TimerParams.Repeat(interval)Repeating, no initial delay, scaled time
.WithDelay(seconds)Set delay before first fire
.WithUnscaledTime(bool)Use Time.unscaledTime instead of Time.time
Handle safety TimerHandle uses generation counters. Cancelling a handle that already fired (or was already cancelled) is a no-op — it never cancels the wrong timer.

AudioPlayer

Pooled AudioSource playback with zero GC allocation per sound. Pre-allocated pool of AudioSource components (default 16). Steal-oldest overflow policy. Handle-based lifecycle. Main-thread only.

Setup

Attach AudioPlayerHost to your persistent GameObject. Optionally create an AudioPlayerSettings ScriptableObject to configure pool size. Access via AudioPlayer.Instance.

Playing sounds

SoundUsage.cs
using Utilities.Audio;

// Simple 2D playback
SoundHandle handle = AudioPlayer.Instance.Play(explosionClip);

// 3D spatial audio at a world position
var p = SoundParams.Default
    .WithVolume(0.8f)
    .WithSpatialBlend(1f)
    .WithPosition(transform.position);

SoundHandle spatial = AudioPlayer.Instance.Play(footstepClip, p);

// Lifecycle control
AudioPlayer.Instance.Pause(handle);
AudioPlayer.Instance.Resume(handle);
AudioPlayer.Instance.Stop(handle);

SoundParams fluent API

Method / PropertyDescription
SoundParams.DefaultVolume 1, pitch 1, no loop, 2D
.WithVolume(float)0–1
.WithPitch(float)Playback speed multiplier
.WithLoop(bool)Loop the clip
.WithSpatialBlend(float)0 = full 2D, 1 = full 3D
.WithPosition(Vector3)World position for 3D audio
.WithMixerGroup(AudioMixerGroup)Route to a specific mixer group

Persistence

Backend-agnostic save/load service. Swap between JSON files and PlayerPrefs via a single settings field — no code changes. Key-based storage with type-safe generics.

Setup

Attach PersistenceServiceHost to your persistent GameObject. Create a PersistenceSettings ScriptableObject via Create → Utilities → Persistence Settings and assign it in the Inspector.

SettingOptionsDescription
BackendJsonFile / PlayerPrefsStorage backend. JsonFile writes to Application.persistentDataPath/<Folder>/<key>.json
FolderstringSubdirectory for JSON files (e.g., "SaveData")
PrettyPrintboolFormat JSON files for readability

Save and load

PersistenceUsage.cs
using Utilities.Persistence;

// Save any [Serializable] type
PersistenceService.Instance.Save("player-progress", playerData);

// Load — returns false if key doesn't exist
if (PersistenceService.Instance.TryLoad("player-progress", out PlayerData loaded))
{
    Apply(loaded);
}

// Check existence, delete, flush PlayerPrefs to disk
bool exists = PersistenceService.Instance.Exists("player-progress");
PersistenceService.Instance.Delete("player-progress");
PersistenceService.Instance.Flush(); // required for PlayerPrefs backend

Reporter

Structured logging with level filtering, tag-based subsystem filtering, and scoped timing backed by Profiler markers. Verbose levels are stripped at compile time in Release builds.

Log levels

Level hierarchy: Trace < Debug < Info < Warning < Error < Off. Trace and Debug require scripting defines to be compiled in. Info, Warning, and Error are always compiled.

Logging.cs
using Utilities.Diagnostics;

// Conditional — compiled only with REPORTER_TRACE define
Reporter.Trace("Packet seq=42", tag: "Transport");

// Conditional — compiled only with REPORTER_DEBUG define
Reporter.Dbg("State machine: Idle → Running", tag: "AI");

// Always compiled in
Reporter.Info("Session started", tag: "Session");
Reporter.Warn("Config missing, using defaults", tag: "Config");
Reporter.Error("Tracking lost for 3.2s", tag: "XR");
Reporter.Exception(ex, context: "LoadConfig", tag: "Config");

Runtime configuration

Bootstrap.cs
// Suppress everything below Warning at runtime
Reporter.Level = ReportLevel.Warning;

// Focus logs on one subsystem
Reporter.TagFilter = "Streaming";

// Or multiple subsystems
Reporter.AddTagFilter("Signaling");
Reporter.AddTagFilter("Streaming");

// Remove all tag filters
Reporter.ClearTagFilters();

// Suppress everything globally
Reporter.Enabled = false;

Scoped timing

Use Reporter.Scope(name, tag) to wrap an operation in a Profiler marker. The timing is recorded in a thread-safe metric registry with zero GC allocation. On the main thread, if Reporter.LogTimings = true, the duration is also emitted as a Trace log line.

Timing.cs
void DecodeMesh()
{
    using var scope = Reporter.Scope("DecodeMesh", "Rendering");
    // ... heavy work ...
} // timing automatically recorded on scope disposal

Compile-time stripping

Add scripting defines in Project Settings → Player → Scripting Define Symbols:

DefineEffectTypical usage
REPORTER_TRACECompiles in Reporter.Trace() call sitesEditor, Dev builds
REPORTER_DEBUGCompiles in Reporter.Dbg() call sitesEditor, Dev builds
(neither)Both stripped from IL entirely — zero costRelease / Shipping
