Reporter API

Namespace: Utilities.Diagnostics. Static class — no singleton. Available to all layers.

Reporter static class
static
static ReportLevel Level { get; set; }
Runtime filter. Messages below this level are suppressed. Default: Trace in Editor, Info in builds.
static
static bool Enabled { get; set; }
Global on/off switch. When false, all output and metric accumulation is suppressed.
static
static bool LogTimings { get; set; }
When true, timing scopes emit a Trace-level log on the main thread. Default: true in Editor, false in builds.
method
static void Trace(string msg, string tag) [Conditional("REPORTER_TRACE")]
High-frequency diagnostic. Zero IL in Release unless REPORTER_TRACE is defined.
method
static void Dbg(string msg, string tag) [Conditional("REPORTER_DEBUG")]
Debug-level. Zero IL in Release unless REPORTER_DEBUG is defined.
method
static void Info / Warn / Error(string msg, string tag)
Always compiled. Filtered at runtime by Reporter.Level.
method
static void Exception(Exception ex, string context, string tag)
Logs with full stack trace preservation.
method
static IDisposable Scope(string name, string tag)
Returns a scoped timing token. Wrap in a using statement — timing is recorded on disposal.
method
static void TagFilter / AddTagFilter(string tag) / ClearTagFilters()
Filter output to specific subsystem tags. null tag filter shows all.
ReportLevel Enum
ValueDescription
TraceHighest verbosity. Requires REPORTER_TRACE define to compile.
DebugRequires REPORTER_DEBUG define to compile.
InfoAlways compiled. General lifecycle events.
WarningAlways compiled. Potential issues.
ErrorAlways compiled. Actionable failures.
OffSilences all output.

Extensibility API

Namespace: Utilities. Types needed to register a custom utility.

[Utility] attribute
param
string name required
Display name shown as the header in Project Settings → Utilities.
param
string description optional
Shown in grey below the header in the panel.
param
Type configType optional
Must derive from UtilityConfig. When set, the editor auto-creates and links a config sub-asset. Its fields appear inline in the panel.
IUtilityInstaller interface
method
void Install(UtilityConfig config, Transform hostRoot)
Called synchronously during BeforeSceneLoad. config is the linked sub-asset or null — always null-guard. Parent host GameObjects to hostRoot; do not call DontDestroyOnLoad on them individually. Installers must have a public parameterless constructor.
UtilityConfig abstract class
base
ScriptableObject
Subclass this to define per-utility settings. Add [SerializeField] fields — they appear inline in the Project Settings panel. Cast config to your subtype inside Install.
