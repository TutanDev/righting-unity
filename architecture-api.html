Layer Attributes

All in namespace Tutan.Architecture. Shorthand attributes inherit from LayerAttribute. All are sealed, non-inherited, single-use.

UtilityAttribute Attribute
ctor
UtilityAttribute()
Marks a type as cross-cutting. Edges targeting this type are ignored by topology validation. Not a layer — a type cannot be both Utility and a layer member.

Behavioral Attributes

Declare message flow and use case entry points. All validated by the MSGBUS and USECASE rule sets.

PublishesAttribute Attribute
ctor
PublishesAttribute(Type messageType)
Declares that this class publishes the given message type. AllowMultiple = true. Only Manager types should use this — other layers trigger a MSGBUS violation.
prop
Type MessageType { get; }
The message type being published.
SubscribesToAttribute Attribute
ctor
SubscribesToAttribute(Type messageType)
Declares that this class subscribes to the given message type. AllowMultiple = true. Engines, ResourceAccess, and Resources must not subscribe.
UseCaseAttribute Attribute
ctor
UseCaseAttribute(string name)
Marks a Manager method as a use case entry point with the given name. Target: Method. The validator traces the call chain from this method and validates USECASE001–003.
prop
string Name { get; }
The use case name. Multiple Managers can share the same name — both appear as entry points in the Validation window.

Editor Tools

Two editor windows for exploring and validating your architecture. Both accessible from the Window → Architecture menu.

WindowMenu PathDescription
ArchitectureSettingsProviderEdit → Project Settings → ArchitectureUnified panel — configure enforcement mode and utility assemblies, then run validation inline. Violations list with severity filter, search, and a detail pane showing the rule's Why and a Fix suggestion. Settings asset is auto-created on first visit.
DependencyGraphWindowWindow → Architecture → Dependency GraphInteractive layer-banded graph. Nodes grouped by layer, edges colored by validity. Filter by layer and kind. Toggle Violations Only. Single-click to ping in Project; double-click to open script.
ValidationWindowWindow → Architecture → ValidationTabbed window — Violations list with rule IDs and source types; Use Cases tab with BFS chains and USECASE results.
CompilationHook Validation runs automatically after every successful compilation when EnforcementMode is not Off. Results appear directly in the Unity Console with clickable links to the violating type.
