com.tutan.architecture

Architecture Examples

Tag Your First Type

Assign any class to a layer using shorthand attributes. The validator uses these to enforce topology rules.

SessionManager.cs 
using Tutan.Architecture;

[Manager]
public class SessionManager
{
    // Business logic for session lifecycle.
    // The [Manager] attribute tells the validator this type
    // lives in the Manager layer.
}

// Shorthand attributes for all five layers:
// [Client], [Manager], [Engine], [ResourceAccess], [Resource]

// Verbose form is also available:
[Layer(ArchitectureLayer.Manager)]
public class AnotherManager { }

Full Call Chain Through Layers

A Client calls a Manager, which orchestrates an Engine and ResourceAccess. Each layer only depends on the one below it.

SessionCallChain.cs 
[Client]
public class LobbyClient
{
    private readonly SessionManager _sessionManager;

    public void OnJoinPressed(string roomCode)
    {
        _sessionManager.JoinSession(roomCode);
    }
}

[Manager]
public class SessionManager
{
    private readonly ValidatingEngine _validator;
    private readonly SessionAccess _sessionAccess;

    public void JoinSession(string roomCode)
    {
        if (_validator.IsValidRoomCode(roomCode))
            _sessionAccess.Connect(roomCode);
    }
}

[Engine]
public class ValidatingEngine
{
    public bool IsValidRoomCode(string code)
        => code.Length == 6 && code.All(char.IsLetterOrDigit);
}

[ResourceAccess]
public class SessionAccess
{
    public void Connect(string roomCode) { /* network call */ }
}

Violation Detection

The validator catches illegal dependencies. Here a Client skips the Manager layer and calls an Engine directly.

Violation.cs 
[Client]
public class GameplayClient
{
    // Violation: Client must not depend on Engine directly.
    // The closed architecture requires Client -> Manager -> Engine.
    private readonly ScoringEngine _engine;
}

[Engine]
public class ScoringEngine
{
    public int Calculate() => 42;
}

// Validate Now outputs:
// [Error] ARCH001: disallowed dependency:
//   GameplayClient [Client] -> ScoringEngine [Engine] via Field
com.tutan.functional

Functional Examples

Safe Component Lookup

Use Optional<T> to safely chain component lookups without null checks. Destroyed Unity objects are automatically treated as None.

HealthReader.cs 
using UnityEngine;
using Tutan.Functional;
using static Tutan.Functional.F;

public class HealthReader : MonoBehaviour
{
    [SerializeField] private GameObject target;

    private void Start()
    {
        Optional<Health> maybeHealth = Some(target)
            .Then(go => go.GetComponent<Health>());

        int hp = maybeHealth
            .Then(h => h.Current)
            .Or(0);

        Debug.Log($"Current HP: {hp}");
    }
}

UI Fallback When Data is Missing

Chain Filter, Then, and Or to build a clean data pipeline for UI display with sensible defaults.

PlayerNameView.cs 
public class PlayerNameView : MonoBehaviour
{
    [SerializeField] private TMP_Text label;

    public void Render(Optional<string> playerName)
    {
        label.text = playerName
            .Filter(name => name.Length > 0)
            .Then(name => $"Pilot: {name}")
            .Or("Pilot: Guest");
    }
}

Result Pipeline with Error Handling

Chain operations with Result<T> and handle all errors once at the boundary. Parse, validate, and apply in a single composable pipeline.

PlayerBootstrap.cs 
public class PlayerBootstrap : MonoBehaviour
{
    [SerializeField] private string configJson;

    public void Initialize()
    {
        Result<PlayerConfig> result =
            Try(() => JsonUtility.FromJson<PlayerConfig>(configJson))
                .Filter(cfg => cfg != null)
                .Then(cfg => Validate(cfg));

        result.Match(
            onError: e => Debug.LogError($"Bootstrap failed: {e.Message}"),
            onSuccess: cfg => Debug.Log($"Player initialized: {cfg.DisplayName}"));
    }

    private Result<PlayerConfig> Validate(PlayerConfig cfg)
    {
        if (string.IsNullOrWhiteSpace(cfg.DisplayName))
            return Error("DisplayName is required");

        return Success(cfg);
    }
}

Safe Resource Loading

Success() automatically returns an error for null references, including destroyed Unity objects, so missing assets surface as Result errors.

EnemyPrefabLoader.cs 
public class EnemyPrefabLoader
{
    public Result<GameObject> LoadEnemy(string path)
    {
        return Success(Resources.Load<GameObject>(path))
            .Then(prefab => Object.Instantiate(prefab));
    }
}

// Usage:
// loader.LoadEnemy("Enemies/Goblin").Match(
//     onError: e => Debug.LogError(e.Message),
//     onSuccess: enemy => PlaceInWorld(enemy));
com.tutan.utilities

Utilities Examples

MessageBus: Pub/Sub

Define a struct message, subscribe in one component, publish from another. The publisher has no reference to subscribers.

TeleportSystem.cs 
// Define a message — must be unmanaged struct
public struct PlayerTeleported : IMessage
{
    public Vector3 Origin;
    public Vector3 Destination;
    public float Duration;
}

// Subscriber — reacts to teleport events
public class FadeController : MonoBehaviour
{
    SubscriptionToken _token;

    void OnEnable()
        => _token = EventBus.Instance
            .Subscribe<PlayerTeleported>(OnTeleported);

    void OnDisable()
        => EventBus.Instance.Unsubscribe(_token);

    void OnTeleported(ref PlayerTeleported msg)
        => StartFade(msg.Duration);
}

// Publisher — no reference to FadeController
public class TeleportManager : MonoBehaviour
{
    public void ExecuteTeleport(Vector3 origin, Vector3 dest, float dur)
    {
        var msg = new PlayerTeleported
            { Origin = origin, Destination = dest, Duration = dur };
        EventBus.Instance.Publish(ref msg);
    }
}

MessageBus: Thread-Safe Enqueue

Enqueue messages from worker threads. They are dispatched safely on the main thread during DrainQueues().

VideoStream.cs 
public struct FrameDecoded : IMessage
{
    public int FrameIndex;
    public int Width, Height;
}

// Called on a network/decode thread — NOT the main thread
public class VideoStreamReceiver
{
    void OnFrameReady(int index, int w, int h)
    {
        var msg = new FrameDecoded
            { FrameIndex = index, Width = w, Height = h };

        // Thread-safe. Dispatched on main thread next frame.
        EventBus.Instance.Enqueue(msg);
    }
}

// Main-thread subscriber — safe to touch Unity API
public class VideoTextureUpdater : MonoBehaviour
{
    SubscriptionToken _token;
    void OnEnable()  => _token = EventBus.Instance
        .Subscribe<FrameDecoded>(OnFrame);
    void OnDisable() => EventBus.Instance.Unsubscribe(_token);

    void OnFrame(ref FrameDecoded msg)
        => UpdateTexture(msg.FrameIndex, msg.Width, msg.Height);
}

Timer: One-Shot and Repeating

Schedule callbacks with handle-based lifecycle. Cancel safely at any time. Supports scaled and unscaled time.

TimerExamples.cs 
// One-shot respawn delay
public class PlayerRespawnController : MonoBehaviour
{
    TimerHandle _handle;

    public void OnPlayerDied()
    {
        _handle = Timer.Instance.Schedule(3f, Respawn);
    }

    void Respawn() { _handle = TimerHandle.Invalid; }
}

// Repeating tick — always cancel in OnDisable
public class CooldownBar : MonoBehaviour
{
    TimerHandle _tick;

    void OnEnable()
        => _tick = Timer.Instance
            .Schedule(TimerParams.Repeat(0.05f), UpdateBar);

    void OnDisable()
    {
        Timer.Instance.Cancel(_tick);
        _tick = TimerHandle.Invalid;
    }
}

// Unscaled time — fires even when Time.timeScale == 0
var p = TimerParams.Default
    .WithDelay(5f)
    .WithUnscaledTime(true);
var handle = Timer.Instance.Schedule(p, OnCountdownDone);

Persistence: Save and Load

Backend-agnostic serialization. Swap between JSON files and PlayerPrefs via a ScriptableObject setting.

SaveSystem.cs 
[System.Serializable]
public class PlayerStats
{
    public int Level;
    public int Gold;
    public float PlayTime;
}

// Save
var stats = new PlayerStats { Level = 5, Gold = 1200, PlayTime = 3600f };
PersistenceService.Instance.Save("player.stats", stats);

// Load
if (PersistenceService.Instance.TryLoad<PlayerStats>("player.stats", out var s))
    Debug.Log($"Welcome back, Level {s.Level}!");

// Check and delete
bool exists = PersistenceService.Instance.Exists("player.stats");
PersistenceService.Instance.Delete("player.stats");

Reporter: Logging and Scoped Timing

Leveled logging with tag filtering and compile-time stripping. Scoped timing measures code blocks with zero allocation.

ReporterExamples.cs 
// Leveled logging — Trace and Debug are stripped from builds
Reporter.Trace("Frame tick", tag: "Loop");
Reporter.Info($"Player joined: {name}", tag: "Session");
Reporter.Warn("Pool running low", tag: "Audio");
Reporter.Error("Connection lost", tag: "Network");

// Tag filtering — show only messages tagged "Network"
Reporter.TagFilter = "Network";

// Scoped timing — zero-allocation, stack-allocated
using var _ = Reporter.Time("DecodeFrame", "Streaming");
// ... expensive work ...
// On dispose: logs elapsed ms and records in MetricRegistry

// Query metrics
var snapshot = Reporter.GetMetric("DecodeFrame");
if (snapshot.HasValue)
    Debug.Log($"Avg: {snapshot.Value.AvgMs:F2}ms");
